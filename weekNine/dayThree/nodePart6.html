Node.js Project

Introduction

So far so good. Am assuming you had a great journey to this point. I surely have enjoyed the Node.js journey. Before bringing a real-life project to life with what we have learned in Node.js, let us take a step back and review what we have grasped.

Take a pause and ask yourself, if someone were to ask me what Node.js is, what would be my answer? Tik tok tik tok.

If it were for me, I would simply state that Node.js is an asynchronous, event-driven, Javascript runtime environment designed to develop scalable network applications . Wow, slow down there and take a break. What do you mean by all this tech jargon you just threw around for me in your definition? Well, let us see what we simply mean by all this.

  1. Asynchronous:- this simply means that two or more processes can be run at the same time without interfering with each other or one waiting for the other.
  2. Event Driven:- this means that the flow of a program or process will be determined by an event taking place.
  3. Javascript runtime environment:- refers to the provision of libraries, tools, or infrastructure that facilitate the running of Javascript code on a computer.
  4. Scalable:- easy to change the size in terms of increasing the size or reducing it.
  5. Network application:- a network application is an application that runs on one host that provides communication to another communication on another application running on a different host.

With this understanding of the Node.js terms used in its definition, how does Node.js fall in delivering all these?

  1. Asynchronous:- any application developed using Node.js will pass async tasks to an event loop together with a callback. This event pool will efficiently manage a thread pool of tasks and execute them efficiently. Then each callback will be executed as the tasks are complete.
  2. Event Driven:- From the asynchronous point above, we have noted that with Node.js an event pool will be created. This means that an event has to take place for a Node.js driven process to take place.
  3. Javascript runtime environment:- Javascript is a programming language. So Node.js will provide the platform upon which the commands are written in JS to run. For example, with Javascript, we have events like click, hover, mousein, and mouseout to mention a few. These events are what trigger Node.js processes.
  4. Scalable:- with Node.js, we can be able to easily grow the side of the program in terms of tasks it handles without affecting any other tasks.
  5. Network application:- With Node.js, one can be able to write Javascript on both the front and back end. This means that a full-stack application can be developed using Node.js. The front end will be an application on one host and the back end will be on a different host. However, with Node.js, the back-end application will be able to communicate with the front-end application without there being the need for a server-side programming language involved.

Well, that's awesome. Imagine the power you could possess when developing applications with such unlimited power. You do not need to think too hard and far about such possibilities. Node.js is here to power such possibilities. Now it is time for the rubber to meet the road by taking up a real-time project. Node version and installations

The very first step is to ensure that we have Node.js installed on our machines. Open your terminal and run the command node.js -v to check the existing version of Node.js. If it does not exist, kindly refer to the official Node.js documentation via: https://nodejs.org/en. It is always advisable to ensure that you are using the latest LTS version of Node.js.

If you have nvm on your PC, you can simply use it to install the desired version of Node.js that you wish to have on your PC. You will simply use the command: nvm install <versionnumber>. Below is an example of an upgrade of node from version 18 to 20 using nvm:
