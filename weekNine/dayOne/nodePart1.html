Node.js is an open source cross-platform runtime environment written in JavaScript. The V8 JavaScript engine in Chrome serves as its foundation, parsing and executing JavaScript code. Because Node employs an event-driven, non-blocking I/O model, it is lightweight and quick. One of the primary causes of Node's rise in popularity is this programming paradigm.

It is called Node because it is used to build simple single-process blocks called nodes. These nodes can be organized with good networking protocols for communication with each other and be scaled up to build large distributed programs.

Think of a node like a small building block of code that can perform a specific task. These modules can communicate with each other using good networking methods, and we can combine them to create more complex and larger programs. It's like building with LEGO blocks where each block (node) has its purpose, and you connect them to build something bigger.

Let us now learn how node.js works.

In Node.js, the big idea is "non-blocking." This means that when you ask Node to do something -(like read a file), it doesn't wait around. Instead:

  • You tell Node what to do by giving it a task (a function).
  • You provide the task with any needed information (arguments).
  • You also give Node a callback function.
  • Node moves on to do other things.
  • When the task is done, Node calls your callback function with the result.

It's like telling someone, "Please go get me a burger, and here's some money. When you have the burger, call me so I can eat it." You don't just stand there waiting for the burger; you can do other stuff while you wait for the call that says, "Burger's ready!" This is what makes Node fast and efficient.

Difference between node.js and vanilla js

  1. The main difference between node.js and vanilla.js is that node runs in the server and not in the browser like vanilla js. This means that we are working on the backend and not just the frontend part of the application.
  2. The console is the terminal window in your code editor and not on the browser like the one we were using for vanilla.js. On the terminal you just type node and it opens up the console for node as shown below.

    3. You may also be able to run javascript in the node console. Add the following code in your server.js file then we will run the code on the terminal starting with node followed by the name of the file. In this case it is the server. 

    console.log(" Hello World")
  3. Another difference between the two is that there is a global object in node instead of a window object.In web development, when you work with JavaScript in a web browser, you often use something called the "window object." It's like a big container that holds information about the web page and lets you do things with it.

    4. Now, when you use JavaScript in Node.js (which is for server-side programming), there's no "window object." Instead, there's a thing called the "global object." It's similar to the "window object," but it's designed for server-side code. Think of it as a container that holds information and tools for your server-side program, like handling files, sending data, and other server-related stuff.In the server.js file console log the global object then run the file on the terminal to be able to see the results. 

    console.log(global)
  4. Node.js uses commonJs modules instead of ES6 modules that we were using in javascript - In JavaScript for web browsers, we often use a module system called "ES6 modules." It's like having special boxes for each part of our code.

    5. Now, when we write code in Node.js (for server-side tasks), we use a different system called "CommonJS modules." These modules work a bit differently from ES6 modules. Instead of using special boxes, we use a different way to organize and share our code. So, it's like choosing a different type of toolbox for the job. When we code for the web browser, we might use one kind of toolbox (ES6 modules), and when we code with Node.js, we use another kind of toolbox (CommonJS modules). They help us keep our code organized and easy to use, depending on where we're working.


Let us look at an example of this.

server.js


    const os = require('os');

    console.log(os.type())
    console.log(os.version())
    console.log(os.homedir())

Let us understand what each line of code does here

  • const os = require('os'); - This line of code is telling Node.js to use a built-in module called os. Think of it as asking Node.js to provide information about the computer that you are using.
  • console.log(os.type()): Here, we're using the os module to find out the type of operating system running on your computer.
  • console.log(os.version()): This line is similar, but it's asking for the version of your operating system. You'll get a version number of your computer.
  • console.log(os.homedir()): Finally, this line is checking for the home directory of the current user on the computer. It's like asking the users main folder on the computer.

Run the node console to be able to see the results on the terminal.
